# SwiftClip Template Catalog

Use this file to separate direct generation templates from external inspiration templates.

- Internal core templates: supported for direct code generation in this repository.
- External references: not directly generated by the builder; used only for recommendation, visual alignment, and brief shaping.

## Recommendation Rules

1. Decide whether the recommendation is internal-core-first or external-reference-first.
2. Prefer internal core templates when the user wants direct automatic generation.
3. Use external references when the user wants style inspiration or planning help.
4. Filter by aspect ratio.
5. Then filter by duration bucket:
   - short: 5s to 8s
   - medium: 9s to 15s
   - long: 16s to 30s
6. Then rank by tag fit and description fit.
7. Return 2-3 results, not the full catalog, unless the user explicitly asks for all references.

## Quick Routing Table

Use this as the first-pass onboarding router before broader recommendation logic.

| user intent | preferred internal base template | fallback | routing reason |
| --- | --- | --- | --- |
| product promo, launch, brand announcement | `ProductLaunch` | `TutorialIntro` | strongest internal base for polished hero-led marketing beats |
| tutorial, course intro, walkthrough opener | `TutorialIntro` | `SaaSPromo` | clean framed intro with a compact copy surface |
| social short, vertical brand post, quick hook | `SocialStory` | none | only internal 9:16 execution base |
| speaker nameplate, event lower-third, broadcast ID | `LowerThird` | `ProductLaunch` | direct match for overlay-style compositions |
| SaaS feature showcase, product capabilities, multi-beat feature story | `SaaSPromo` | `ProductLaunch` | strongest internal base for multi-scene feature storytelling |

If a user intent maps cleanly here, recommend the preferred internal base first and ask only the minimum follow-up questions needed to complete the brief.

## External Reference Mapping Guidance

When the user picks an external reference, map it to the closest internal core template before code generation.

| external reference pattern | preferred internal base | mapping note |
| --- | --- | --- |
| Brand, logo, intro sting | `ProductLaunch` or `LowerThird` | choose `LowerThird` for minimal identity overlays; otherwise `ProductLaunch` |
| CTA, outro, end card | `TutorialIntro` or `ProductLaunch` | use `TutorialIntro` for compact card framing |
| Vertical social inspiration | `SocialStory` | keep 9:16 output and simplify copy surface |
| SaaS feature showcase | `SaaSPromo` | best internal base for multi-scene feature narratives |
| Tutorial or education pacing | `TutorialIntro` | best internal base for guided intros |
| Broadcast lower-third or speaker overlay | `LowerThird` | direct match |

## Guidance For External References

- Recommend them normally.
- Produce a brief and scene plan.
- If code generation is requested, first remap them to one of the 5 internal core templates.
- Do not promise direct generation from an external reference.
